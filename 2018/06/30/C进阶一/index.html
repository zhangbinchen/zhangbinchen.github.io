内存

内存分区

  • 数据类型：

    为了更好的进行内存的管理，让编译器能确定分配多少内存
    类型是对数据的抽象
    类型相同的数据具有相同的表示形式、存储格式及相关操作
    程序中的所有数据都必定属于某种数据类型
    数据类型可以理解为创建变量的摸具：固定大小内存的别名

  • 数据类型别名（typedef）
  • typedef
  1. 起别名-简化struct关键字
  2. 区分数据类型
  3. 提高代码移植性
  • void 数据类型

    不可以利用void创建变量（无法给无类型变量分配内存）
    用途：限定函数返回值，函数参数
    void* 万能指针 可以不通过强制类型转换就转成其他类型的指针

  • sizeof 用法

    本质:是一个操作符，不是函数
    返回值类型 unsigned int
    用途：统计数组长度

  • 变量的修改方式：

    直接修改
    间接修改（指针）

内存分区模型

  1. 程序运行前
  • 代码区（共享、只读）
  • 数据区（存放数据：全局变量、静态变量、常量）

    已初始化数据区：data
    未初始化数据区：bss

  1. 程序运行后
  • 栈stack（符合先进后出数据结构，编译器自动管理分配和释放，有限容量）

    栈是一种先进后出的内存结构，由编译器自动分配释放，存放函数的参数值，返回值，局部变量等。在程序运行过程中实时加载和释放，因此局部变量的生存周期为申请到释放该段栈空间

【注意】：不要返回局部变量的地址，局部变量在函数执行后就被释放了，释放的内存没有权限操作了，如果操作结果未知。

  • 堆heap（容量远远大于栈，不是无限。手动开辟malloc（），手动释放free（））

    堆是一个大容器，它的容量要远远大于栈，但没有栈那样先进后出的顺序。用于动态内存分配，堆在内存中位于bss区和栈区之间。一般由程序员分配和释放，若程序员不释放，程序结束时由操作系统回收。

【注意】：主调函数没有分配内存，被调函数需要用更高级的指针去修饰低级指针，进行内存分配

static和extern区别

  • static修饰的静态变量：
  1. 在运行前分配内存，程序运行结束，生命周期结束
  2. 在本文件内都可以使用
  • extern：可以提高变量的作用域
    【-_-】:在c语言中全局变量前都有隐藏了的extern修饰符

常量

  • const修饰的变量（常量）
  1. const修饰的全局变量：放在常量区，不能修改（直接修改失败，间接修改，语法通过，运行失败）
    【原因】：受到常量区的保护-_-
  2. const修饰的局部变量：放在栈区，可间接修改。（直接修改失败）并非真正的常量，伪常量，不可以用作数组的初始化
  3. 字符串常量：

    vs 下将多个相同的字符串常量看成一个
    不可以修改字符串常量
    ANSI并没有制定出字符串是否可以修改的标准，根据编译器不同，可能最终的结果也是不同的
    【-_-】字符串常量地址是否相同

    • tc2.0，同文件字符串常量地址不同
    • vs2013，字符串常量地址同文件不同文件都相同
    • Dev c++、QT同文件相同，不同文件不同。

【-_-】:

  • 数据区包括：堆、栈，全局/静态存储区
  • 全局/静态存储区包括：常量区，全局区，静态区
  • 常量区包括：字符串常量区，常变量区
  • 代码区：存放程序编译后的二进制代码，不可寻地址区

宏函数

函数调用流程

  • 栈容器在程序运行中具有极其重要的地位。最重要的，栈保存一个函数调用所需要维护的信息，这通常被称为栈帧（Stack Frame）或者活动记录（Activate Record）.一个函数调用过程所需的信息一般包括以下几个方面：

    函数的返回值
    函数的参数
    临时变量
    保存的上下文：包括在函数调用前后需要保持不变的寄存器

  • 流程：局部变量、函数形参、函数返回地址、入栈和出栈

调用惯例

【主调函数和被调函数必须要有一致的约定，才能正确的调用函数，这个约定我们称为调用惯例】

  1. 调用惯例包含内容：出栈方、参数传递顺序、函数名称修饰
  2. c/c++下默认调用惯例：cdecl 从右到左，主调函数管理出栈
  • 函数参数的传递顺序和方式

    函数传递方式有很多方式，最常见的是通过栈传递，函数的调用方将参数压入栈中，函数自己再从栈中将参数取出，对于有多个参数的函数，调用惯例要规定函数调用方将参数压栈的顺序：从左到右还是从右到左。有些调用惯例还允许使用寄存器传递参数，以提高性能。

  • 栈的维护方式
    在函数将参数压入栈中之后，函数体会被调用，此后需要将被压入栈中的参数全部弹出，一时的栈在函数调用前后保持一致，这个弹出的工作可以由函数的调用方来完成，也可以由函数本身来完成。为了在链接的时候对调用惯例进行区分，调用惯例要对函数本身的名字进行修饰。不同的调用惯例有不同的名词修饰策略
    事实上，在c语言中，存在着多个调用惯例，而默认的是cdecl惯例。比如我们对于某个函数的声明，完整写法应该是：int_cdecl func(int a,int b);
    【-_-】:_cdecl不是标准关键字，在不同的编译器里可能有不同的写法，例如：gcc里就不存在_cdecl这样的关键字，而是使用_attribute_((cdecl))
调用惯例 出栈方 参数传递 名字修饰
cdecl 函数调用方 从右至左参数入栈 下划线+函数名
stdcall 函数本身 从右至左参数入栈 下划线+函数名+@+参数的字节数
fastcall 函数本身 前两个参数由寄存器传递，其余参数通过堆栈传递 @+函数名+@+参数的字节数
pascal 函数本身 从左至右函数参数入栈 较为复杂，参见文档
