include头文件包含的两种不同写法及其含义

#include<> 与 #include””

  • <>表示系统直接按系统指定的目录检索
  • “”表示系统先在””指定的路径（没写路径代表当前路径）查找头文件，如果找不到，再按系统指定的目录检索

gcc 编译的4步骤：

  • gcc hello.c -o hello.exe (包含四步)
  • gcc -E hello.c -o hello.i 预处理：展开头文件、展开宏定义，替换注释，不检查语法
  • gcc -S hello.i -o hello.s编译：逐行检查语法错误，将c程序翻译成汇编指令，生成.s文件
  • gcc -c hello.s -o hello.o(目标文件) 汇编：将指令翻译成二进制机器码
  • gcc hello.o -o hello.exe 链接：连接程序所使用的库

vs2013常见c4996错误

微软在vs2013中不再建议使用C的传统库函数 scanf、strcpy、sprintf等，直接使用会提示C499错误
解决办法：

  • 在程序的首行添加 #define _CRT_SECURE_NO_WARNINGS
  • 在程序中添加 #pragma warning(disable:4996)
