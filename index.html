    C和C++中struct的区别

    • c语言中struct只有变量
    • c++中struct 既有变量，也有函数

    类的封装

    1. 属性和变量合成一个整体
    2. 给属性和函数增加访问权限
    • 封装：
      1. 把变量（属性）和函数（操作）合成一个整体，封装在一个类中
      2. 对变量和函数进行访问控制
    • 访问权限
      1. 在类的内部（作用范围内），没有访问权限之分，所有成员可以互相访问
      2. 在类的外部（作用范围外），访问权限才有意义：public，private,protected
      3. 在类的外部，只有public修饰的成员才能被访问，在没有涉及继承与派生时
        【private和protected是同等级的，外部不允许访问】
        |访问属性|属性|对象内部|对象外部|
        |:—-:|:—–:|:—-:|:—-:|
        |public|公有|可访问|可访问|
        |protected|保护|可访问|不可访问|
        |private|私有|可访问|不可访问|

    struct和class的区别

    class默认访问权限为private，struct默认访问权限为public

    将变量设为private

    1. 可赋予客户端访问数据的一致性
    2. 可细微划分访问控制
      【尽量将成员属性设置为私有】
    • 好处：

      自己可以控制读写权限
      可以对设置内容加有效性验证

    C++概述

    • c++是对c的扩展，是c语言的超集。
    • 库：是编程模块的集合，可以在程序中调用他们。库对很多常见的编程问题提供了可靠的解决办法，因此可以节省程序员大量的时间和工作量。
    • c++在c的基础上添加了面向对象编程和泛型编程的支持，并且继承了c语言的高效、简洁，快速和可移植的传统。
    • c++融合了3种不同的编程方式：

      -c语言代表的过程性语言
      -c++在c语言基础上添加的类代表的面向对象的语言
      -c++模板支持的泛型编程。
      【ANSI在1998年定制了c++第一套标准】

      面向过程

      【面向过程思想核心】：功能分解，自顶向下，逐层细化（程序=数据结构+算法）

      面向对象

    • 在面向对象中，算法与数据结构被看作一个整体，称作对象
    • 对象=算法+数据结构
    • 程序=对象+对象……
      【面向对象三大特性】
    1. 封装
    2. 继承
    3. 多态

    C++对C的扩展

    1. ::作用域用算符

    作用域运算符可以用来解决局部变量与全局变量的重名问题，在局部变量作用域内，可以用::对
    被屏蔽的同名全局变量进行访问（::前什么都不加表示全局作用域）

    1. 名字控制
    • c++允许我们对名字的产生和名字的可见性进行控制
    • c++中我们通过命名空间控制对名字的访问（namespace）

      namespace

    • 用途：解决名称冲突
    • 命名空间下可存放：变量、函数、结构体、类……
    • 命名空间必须要声明在全局作用域
    • 命名空间可以嵌套命名空间
    • 命名空间是开放的，可以随时将新成员添加到命名空间下
    • 命名空间可以匿名
    • 命名空间可以起别名
    • 命名空间的声明和实现可分离

    2.1. using声明

    • using声明可使指定的标识符可用
    • 当using声明与就近原则同时出现，出错，尽量避免
      2.2. using编译指令
    • 当using编译指令与就近原则同时出现，优先使用就近
    • 当using编译指令有多个，需要加作用域区分
    • 【注意】使用using声明或 using编译指令会增加命名冲突的可能性，也就是说，如果有命名空间，并在代码中使用作用域解析运算符，则不会出现二义性。
    1. c++增强部分
    • 全局变量检测增强 c++严格检测出重定义
    • 函数检测增强，返回值检测、形参类型检测、函数调用参数个数
    • 类型转换检测曾强；
    • struct（结构体）增强 （c中不可以有函数必须有关键字struct）c++可以放函数、创建结构体变量，可以简化关键字struct
    • bool类型扩展（bool类型占一个字节大小）（c语言下没有bool类型 c++有bool类型）true（1） false（0）
    • 三目运算符增强（a<b?a:b）=100;c++下表示给三目运算结果赋值。
    • c语言三目运算表达式返回值为数据值，为右值，不能赋值
    • c++三目运算表达式返回值为变量本身（引用）为左值，可以赋值
      【左值和右值的概念】
      在c++中可以放在赋值操作符左边的是左值，可以放到操作赋值符右面的是右值。有些变量既可以当左值又可以当右值
      左值为：Lvalue，L代表Location表示内存可以寻址，可以赋值
      右值为：Rvalue，R代表Read，就是可以知道它的值
      int temp=10；temp在内存中有地址，10没有，但是可以Read到它的值
    • const增强

      全局const受到常量区保护，运行修改失败（c、c++结论相同）
      局部const,c++下const修饰的变量称为常量，可以用来初始化数组（c语言中const是伪常量，不可以初始化数组分配到栈上）

    • c语言中全局const会被存储到制度数据段，c++中全局const当声明extern或者对变量取址时，编译器会分配存储地址，变量存储在只读数据段，两个都受其到了保护不可修改
    • c语言中局部const存储在堆栈区，只是不能通过变量直接修改const制度变量的值，但是可以跳过编译器的检查，通过指针间接修改该const值
    • c中const默认为外部链接，c++中const默认为内部链接，当c语言两个文件中都有const int a的时候编译器会报重定义错误。而在c++中就不会，因为c++中的const默认是内部链接如果想让c++中的const具有外部链接属性，必须显示声明为:extern const int a=10;
      【c语言】

      全局const 直接修改失败，间接修改语法通过，运行失败
      局部const 直接修改失败，间接修改成功

    【c++】

    全局const 直接修改失败，间接修改语法通过，运行失败
    局部const 直接修改失败，间接修改失败
    c++中const修饰的可称作常量

    • 指针变量
    • 指针是一种数据类型，占用内存空间，用来保存内存地址

    野指针和空指针

    • 空指针：不能向NULL或者非法内存拷贝数据
    • 野指针：
      1. 指针变量未初始化：
        任何指针变量刚被创建的时候不会自动成为NULL指针，他的缺省值是随机的，他会乱指一气，所以，指针变量在创建的同时应当被初始化，要么将指针设为NULL，要么指向合法内存
      2. 指针释放后未置空:
        有时指针在free或delete后未赋值NULL，便会使人们认为是合法的，不要被free和delete的名字迷惑，他们只是将指针所指的内存给释放掉，但并没有把指针本身干掉，此时指针指向的就是“垃圾”内存，释放后的指针应立即将指针置为NULL，防止产生野指针
      3. 指针操作超越变量作用域
        不要返回指向栈内存的指针或引用，因为栈内存在函数结束时会被释放
        【规避野指针】
        ＞1. 初始化时置NULL
        ＞2. 释放时置NULL

    空指针可以重复释放、野指针不可以重复释放

    指针的步长

    • +1后跳跃的字节数
    • 解引用解出的字节数
    • 自定义结构体步长

      通过offsetof（结构体名称，属性）找到属性对应的偏移量
      offsetof引入头文件：#include<stddef.h>

      指针的件接赋值

    • 三大条件

      一个普通变量+指针变量（实参+形参）
      建立关系
      通过*操作内存

    • 利用qt操作地址修改内存

    指针做函数参数的输入输出特性

    • 输入特性：
    1. 在主调函数中分配内存，被调函数使用
    2. 分配在栈上和堆区
    • 输出特性：
    1. 在被调函数中分配内存，主调函数使用

    vim的三种模式

    vim的三种模式：命令模式、文本编辑模式、末行模式
    在末行模式可以直接执行shell命令（：！shell）

    • 命令模式
    快捷键 操作
    保存退出
    ZZ（大写） 保存退出
    代码格式化
    gg=G 代码格式化
    光标移动
    h 光标左移
    j 光标下移
    k 光标上移
    l 光标右移
    w 光标移动一个单词
    gg 光标移动到文件的开头
    G 光标移动到文件的末尾
    0 光标移动到行首
    $ 光标移动到行尾
    nG 行跳转，如：12G-跳转到12行
    删除
    x（小写） 删除光标后一个字符相当于Del
    X（大写） 删除光标前一个字符，相当于Baskspace
    dw 删除光标开始的位置的字，包含光标所在的字符
    d0 删除光标前本行所有内容，不包含光标所在字符
    D[d$] 删除光标后本行所有内容，包含光标所在的字符
    dd 删除光标所在的行（本质其实就是剪切）
    ndd 从光标当前行向下删除指定的行数，如：15dd
    v/ctrl+v 使用h、j、k、l、移动选择内容然后按d删除【ctrl+v是列模式，块选择】
    撤销和反撤销
    u 一步一步撤销，相当于ctrl+z
    ctrl+r 反撤销，相当于word中的ctrl+y
    复制粘贴
    yy 复制当前行
    nyy 赋值n行，如：10yy
    P（大写） 在光标所在位置向上新开辟一行，粘贴
    p小写 在光标所在位置向下新开辟一行，粘贴
    剪切 按dd或ndd删除，实际将其剪切到了剪切板中，p粘贴就行
    v/ctrl+v 使用h、j、k、l、移动选择内容然后按d删除【ctrl+v是列模式，块选择】，y复制，P/p光标前/后粘贴
    替换操作
    r 替换当前字符
    R 替换当前光标后的字符
    查找
    / /xxxx，光标位置开始搜索，按n向下，N向上
    ？ ？xxxx，将光标移动到带搜索的字符串上，然后按n向上，N向下
    # 将光标移动到带搜索的字符串上，然后按n向上搜索，N向下
    shift+k 在带搜索的字符串上按shif+k，或者K（大写）可以查看相关的帮助文档
    • 从命令模式进入编辑模式
    快捷键 操作
    i 在光标前插入
    a 在光标后插入
    I 在光标所在行的行首插入
    A 在光标所在行的行尾插入
    o（小写） 在光标所在的行的下面创建一行，行首插入
    O（大写） 在光标所在行的上面创建一行，行首插入
    s（小写） 删除光标后面的字符，从光标当前位置插入
    S（大写） 删除光标所在当前行，从行首插入
    块插入（列模式） ctrl+v进入快选择，shift+i向前插入字符，按两下esc完成
    • 末行模式（：）
    快捷键 操作
    保存退出
    q 退出
    q! 强制推出，不保存修改内容
    w 保存修改不退出
    wq 保存退出
    x 相当于wq
    替换操作
    ：s/old/new/ 光标所在行的第一个old替换为new
    ：s/old/new/g 光标所在行所有old全部替换为new
    ：m，ns/old/new/g 将第m行至第n行之间的old全部替换为new
    ：%s/old/new/g 当前文件的所有old替换为new

    • 字符删除
      • ctrl+h(Backspace)删除光标前边的字符
      • ctrl+d删除光标后边的字符
      • ctrl+u删除光标前所有内容
      • ctrl+k删除光标后所有内容

    Linux 下的目录结构

    • 倒树状 根目录用“/”
    主要目录简介
    • /bin（binary，二进制文件，存放可执行程序，shell命令，如：ls,rm,等命令存放目录）
    • /sbin（Super User，存放系统管理员使用的系统管理程序，如ifconfig，halt，shutdown，reboot等系统命令）
    • /dev（device，设备文件，Linux下一切皆文件，硬件鼠标键盘文件，字符设备块设备文件……）
    • /lib（动态库文件）
    • /mnt（手动挂载目录）
    • /media（外设的自动挂载目录，如光驱）
    • /root（root用户家目录）
    • /usr（unix system resource /usr/include目录存放头文件）
    • /etc(存放配置文件:etc/passwd、etc/group、etc/profile（系统的配置文件，修改该文件会影响这个系统下面的所有用户）)
    • /opt（安装第三方应用程序，如oracle数据库）
    • /home（Linux操作系统所有用户的家目录（家目录又叫宿主目录或者主目录））
    • tmp（存放临时文件，存放在这个目录下的文件会在系统重新启动后自动清除）
      【”.”表示当前目录】
      【”..”表示当前目录上一级目录】

    文件和目录操作命令

    • ls（查看指定目录下的文件信息）

      ls -a:列出当前目录下的所有文件
      ls -R:递归方式列出所有目录中的内容
      ls -l:列出文件的详细信息（详细信息包含7部分内容）
      ls -ltr:按照时间递归排序显示详细信息

      1. 第一个字符:表示文件类型
        • -:普通文件
        • d:目录文件
        • l:软链接（符号链接），相当于windows中的快捷方式
        • s:socket套接字
        • p:管道
        • b:块设备（磁盘）
        • c:字符设备（键盘）
      2. 第2-10个字符：用户操作权限
        • 234字符：文件所有者对文件的操作权限
        • 567字符：文件所属组用户对文件的操作权限
        • 8910字符：其他人对文件的操作权限
      3. 硬链接计数
        • 对目录而言，链接计数等于该目录下所有目录总数，但不包括该目录的自目录下的目录文件数量
        • 对于文件而言，指的是该文件所有的硬链接文件数量
      4. 文件所有者：
      5. 文件所属组
      6. 文件大小（目录文件指的是是目录大小即4k）
      7. 文件创建时间或最后修改时间

    • cd（切换目录）

    • pwd（查看当前工作目录）
    • which（显示命令所在目录，如which ls）
    • touch（文件不存在创建空文件，文件存在更新文件的最后修改时间）
    • mkdir（创建新目录，可创建多级目录加参数：-p）
    • rmdir（删除空目录）
    • rm（删除文件：-r 递归删除目录 -i提示用户是否删除 -f:强制删除）
    • 文件删除不易恢复，操作需谨慎
    • cp（拷贝命令，用法：cp 源 目标）

    • cp命令需注意几种情况：

      1. 若有目录的拷贝需要使用 -r参数
      2. cp file1 file2(不存在):创建file2，再将file1拷贝到file2中
      3. cp file1 file2（存在）：file1覆盖file2
      4. cp -r dir1 dir2（存在）：将dir1在内的目录拷贝到dir2中
      5. cp -r dir1 dir2（不存在）：创建dair2，将dir1中的内容拷贝到dir2，不包括dir1
      6. cp拷贝目录也可以用-a参数，这样可以保留被拷贝的文件的一些属性信息

    • mv(移动文件或者文件重命名：mv 源 目标)

      1. 重命名时若源文件和目标文件均存在时：实际将源文件改名为目标文件且源文件覆盖目标文件
        2.移动：目标一定是目录；若源也是目录：那么将源移动到目录中以子目录存在
    • cat（将文件内容一次性输出到终端若文件太长无法全部显示）
      cat可用于文件重定向：cat file1>file2,相当于cp file1 file2
    • more（文件内容分页显示到终端，只能向下浏览不能回退 ：more+文件名；下一行:回车 下一页：空格 退出 q）
    • less（将文件分页显示到终端，可上下浏览）
      显示下一行：回车、ctrl+p、↓
      显示上一行：ctrl+n、↑
      显示下一页：空格、PageDown
      显示上一页：PageUp
      推出：q
    • head（文件开头起查看前n行：head -n（行数） 文件名；默认显示前10行）
    • tail（从文件尾部向上查看最后n行内容：tail -n（行数）文件名，默认10行）
      【tail显示日志：tail -f test.log(一个终端tail -f test.log，另一个终端：echo”hello world”>>test.log)】
    • ln -s（软链接（类似windows的快捷方式）；文件目录均可设置软链接）
      【注意：ln -s创建软链接要用绝对路径；软链接文件大小为：路径+文件名的总字节数】
    • ln（硬链接：文件名 硬链接名）
      1. 硬链接不能建在目录上
      2. 硬链接不能跨文件系统：硬链接文件和源文件的inode是相同的，文件系统的inode要求唯一，跨文件系统可能会使inode不同
      3. 硬链接的本质：是不同的文件名所在的inode节点是相同的，相同的inode节点指向了相同的数据块，所以他们的文件内容是一样的，文件内容会同步
      4. 硬链接可以起到同步文件的作用：修改file内容其他硬链接文件会同步修改
      5. 删除文件的时候，只要硬链接计数部位0，不会真正删除，起到保护文件的作用
    • wc（显示文件行数，字节数，单词数）

      wc -l file 显示文件的总行数
      wc -c file 显示文件的总字节数
      wc -w file 显示文件的总单词数
      wc file 显示文件的总行数，单词数，总字节数

    • whoami（显示当前登陆的用户名）

    用户、用户组、用户权限

    • chmod（修改文件权限）
    文字设定法：chmod[who][+或-或=][mode]文件名
    操作对象[who] 操作符【+、-、=】 权限【mode】
    u—(user)用户、 +:添加权限 r—读
    g—（group）用户组（同组用户） -：取消权限 w—写
    o—(other)其他用户、 =：赋予给定权限并取消其他权限 x—执行
    a—(all)所有用户【默认】
    示例： 给文件file.txt的所有者和所属组添加读写权限 chmod ug+wr file.txt
    数字设定法：chmod [+、-、=(可省略)][mode]文件名
    0 没有权限（-）
    1 执行权限（x）
    2 写权限（w）
    4 读权限（r）
    示例：给file.txt设置rw-rw-r– chmod 664 file.txt
    注意 使用数字设定法一定要使用3位的8进制数（如：066）
    • chown （修改文件所有者和所属组）

      chown 文件所有者 文件名：sudo chown me file.txt
      chown 文件所有者：文件所属组 文件名：sudo chown me:/. me file.txt
      注意：sudo adduser 用户名（创建新用户）

    • chgrp（修改文件所属组）

      chgrp 用户组 文件或目录名：sudo chgrp me file.txt(将file.txt所属组改为me)

    find

    • 按文件名查询：find 路径 -name “文件名”

    • 按文件类型查询：find 路径 -type 类型

      文件类型（type）：
      普通文件用f表示而不是”-“;
      d—–目录（dir）
      l—–软链接（符号链接）
      b—–块设备文件
      c—–字符设备文件
      s—–socket文件（套接字）
      p—–管道文件

    • 按文件大小查询：find 路径 -size 范围

      范围：大于：+ ，+100k；”+”小于：-，-100k；等于：不加符号：100k
      大小：M（大写）、k（小写）、c（表示字节数）
      例如：查询家目录下大于50k，小于100k的文件：find ~/-size+50k-100k

    • 按文件日期查询：

      创建日期：-ctime -n/+n（n天以内/n天以外）
      修改日期：-mtime -n/+n
      访问日期：-atime -n/+n

    • 按深度

      -maxdepth n(层数)：搜索n层以下的目录，搜索的层数不超过n层
      -mindepth n(层数)：搜索n层以上的目录，搜索的层数不超过n层

    • 高级查找：

      1. find 路径 查找规则 -exec shell命令 {}\;（{}中不能有空格）
      2. find 路径 查找规则 -ok shell命令 {}\;(ok比较安全，在执行操作前询问是否确认操作)
      3. find 路径 查找规则 | xargs shell命令
        如：查找当前目录下所有目录，列出目录中文件详细信息
      4. find ./ -type d -exec ls -l {}\;
      5. find ./ -type d -ok ls -l {}\;
      6. find ./ -type d |xargs ls -l

    grep

    • grep -r（有目录）”查找的内容”搜索的路径

      -r :若是目录，则递归搜索
      -n :显示查到内容所在行号
      -i :忽略大小写查找
      -v :不显示含有某字符串
      如：搜索当前目录下包含helloworld字符串的文件
      grep -r -n-i “HELLo world” ./

    • find 和 grep结合使用：find 查找路径 查找规则 |xargs grep 参数 “查找内容”

      如：find . -name”*.c”|xargs grep -n “main”

    Linux 下常用压缩工具

    1. tar 工具
    参数 作用
    z: 用gzip来压缩/解压缩文件
    j: 用bzip2来压缩/解压缩文件
    c: create，创建新的压缩文件，与x互斥使用
    x: 从压缩文件中释放文件，与c互斥使用
    v: 详细报告tar处理的文件信息
    f: 指定压缩文件的名字
    t: 查看压缩包中有哪些文件
    =============== =========================================
    gzip和bzip2 不能压缩目录，只能一个一个文件进行压缩，压缩后会使原文件消失
    gzip * 压缩当前目录下所有的文件，但是目录不能压缩
    gunzip * 解压当前目录下所有的.gz文件
    bizip2 * 压缩当前目录下所有的文件，但是目录不能压缩
    bunzip2 * 解压当前目录下所有的.bz2文件
    • 压缩：
      • tar cvf 压缩包的名字.tar 要打包压缩的文件或目录
      • tar zxvf 压缩包名字.tar.gz 要打包压缩的文件或目录
      • tar jcvf 压缩包的名字.tar.bz2 要打包压缩的文件或目录
    • 解压缩：
      • tar xvf xxxx.tar.gz
      • tar zxvf xxxx.tar.gz
      • tar jxvf xxxx.tar.bz2
    • 解压到指定目录：-C（大写）【tar zxvf xxx.tar.gz -C 路径】
    • 查看压缩包中有哪些文件：tar -tvf xxx.tar
    1. rar 工具（需要安装）
    • 压缩：
      • rar a -r 要压缩的文件或目录（-r参数表示递归用来压缩目录）
      • 如：rar a -r my aa bb dir(将aa、bb、dir、压缩到my.rar文件中)
      • 压缩新文件无需指定后缀
    • 解压缩：
      • rar x xxx.rar 解压目录
    1. zip工具
    • 压缩：
      • zip -r 压缩包名 要压缩的文件或目录
      • 如：zip -r xxx file dir(将file、dir压缩成xxx.zip文件)
    • 解压缩：

      unzip 压缩包名 -d 解压目录（若解压目录不存在则会自动创建）
      如: unzip xxx.zip -d /home/me/test/first

    gcc编译器

    编译命令 生成文件 操作内容
    gcc -E XXX.c XXX.i 预处理器（cpp）：头文件展开，宏替换，去掉注释和空行
    gcc -S XXX.i XXX.s 编译器（gcc）：源文件编译成汇编文件
    gcc -c XXX.s XXX.o 汇编器（as）：将汇编文件编译成二进制文件
    gcc XXX.o a.out 连接器（ld）：将函数库中相应的代码组合到目标文件中

    gcc常用参数

    参数 功能
    -v 查看gcc版本号，–version也可以
    -E 生成预处理文件
    -S（大写） 生成汇编文件
    -c（小写） 只编译，生成.o文件，通常称为目标文件
    -I 指定头文件所在路径
    -L 指定库文件所在路径
    -l 指定库的名字
    -o 指定生成的目标文件的名字
    -g 包含调试信息，使用gdb调试需要添加-g参数
    -On n=0~3 编译优化，n越大优化的越多
    -Wall 提示更多警告信息
    -D 编译时定义宏

    库的制作和使用

    • 库是二进制文件，是源码我呢见的另一种表现形式，是加了密的源代码；是一些功能相近或者是相似的函数的集合体
    • 使用库的好处:
      提高代码的复用性，提高程序的健壮性
      减少开发者的代码开发量，缩短开发周期
    • 库的使用
      头文件（包含了库函数的声明）
      库文件（包含了库函数的代码实现）
      【注意：】库不能单独使用，只能作为其他执行程序的一部分完成某些功能，也就是说只能被其他程序调用才能使用

    静态库（static library）

    • 制作
      1. gcc -c *.c（源） 分别生成 .o文件
      2. ar rcs lib静态库名.a .o文件
      3. 静态库的命名：lib+自定义库名.a
    • 使用
      gcc 调用库的.c -L要连接的库所在路径 -I头文件所在路径 -l静态库名（去掉前缀lib和后缀.a）-o 目标名

    动态库（共享库 shared library）

    • 制作
      1. gcc -fpic -c *.c(源) 生成.o 文件（-fpic创建与地址无关的编译程序）
      2. gcc -shared *.o -o lib动态库名.so
    • 使用
      1. gcc 调用库的.c -L要连接的库所在路径 -I头文件所在路径 -l动态库名（去掉前缀lib和后缀.so）-o 目标名
      2. 配置依赖的库的绝对路径
        export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:库路经（可临时执行生效一次，若要永久有效则需将此行写入~/.bashrc中或者系统配置文件/etc/profile中，或者把库拷贝到系统库/lib中或者/usr/lib中；以上建议写入当前用户 ~/.bashrc配置文件中执行生效）
