  • 指针

    2018-05-18 |
    • { C语言 }

    概念：

    • 内存地址：内存区的每一个字节都有一个编号，这个编号就是内存地址
    • 指针就是内存地址（指针是内存单元的编号，指针变量是存放地址的变量）

    指针变量的定义和使用

    • 指针也是一种数据类型，指针变量也是一种变量。
    • 指针变量指向谁，就把谁的地址赋值给指针变量
    • “*”操作符操作的是指针变量指向的内存空间
    • 注意：&可以取得一个变量在内存中的地址，但是，不能取及寄存器变量，因为寄存器变量不再内存里，而在cpu里面，所以是没有地址的。

    指针大小：

    • 指针的大小：指针的大小与类型无关，只与平台架构有关。32位系统为4字节，64位系统为8字节
    • sizeof(int )==sizeof(char )==sizeof(long )==sizeof(longlong
      在32位平台所有指针都是32位（4字节）
      在64位平台，所有指针都是64位（8字节）

    通过指针间接修改变量的值

    • p的含义，将p变量的内容取出来，当成地址看待，找到该地址。入托\P在=左侧，指代该空间，如果在=右侧，取出该变量的值

    野指针

    • 指针变量也是变量，是变量就可以任意赋值，不要越界即可（32位为4字节，64位为8字节），但是，任意数值赋值给指针变量没有意义，因为这样的指针就变成了野指针，此指针指向的区域是未知（操作系统不允许操作此指针指向的内存区域）。所以，野指针不会直接引发错误，操作野指针指向内存区域才会出现问题。
    • 定义，但未指定有效地址的指针
    • 编程时务必避免出现野指针

    空指针

    • 野指针和有效指针变量保存的都是数值，为了标志此指针变量没有指向任何变量（空闲可用），c语言中可以把NULL赋值给此指针，这样就标志此指针为空指针，没有任何指针。
    • int p=NULL; NULL：#define NULL（（void ）0）
      在指针刚刚定义之初，指定指针为 NULL;
      在明确指针不再使用的地方。手动置为NULL;

    泛型指针 （void *）

    • ！=void
    • 泛型指针可以接受任意数据类型的地址
    • void *指针可以指向任意变量的内存空间
    • 泛型指针不可以直接间接引用读取数据。必须要强转为具体数据类型

    const修饰的指针变量

    1. const int p
      p可改变       p不可改变
    2. int const P
      p可改变       p不可改变
    3. int const p；
      p不可以改变       p可以改变
    4. const int const p
      p不可以改变       p不可以改变
    • const向右，先知道谁，谁就不能改变
    • 通常用于函数形参修饰，限制参数不可改变（传入参数）

    指针和数组

    • 数组名是地址常量。不能被赋值但可以赋值给指针变量

    指针操作数组(代码如上)

    int arr[]={0,1,3,4,5,5,6,88,9};
    int p=arr;
    结论：p[i]==arr[i]==    (arr+i)==*（p+i）
    p、arr在读数组元素过程中，不会变化

    • 指针和数组名的区别：
    1. 指针是变量，可以赋值。数组名是常量，不能被赋值
    2. sizeof（p）获取的是指针大小，sizeof(数组)得到数组的实际大小

    指针的算术运算

    普通变量中的应用
    • 指针乘除：无意义。error！！！
    • 指针加减整数：

      指针计算不是简单的整数相加
      指针+1，实际加sizeof（指针前面数据类型）的大小
      如果是一个int，+1得结果是增加一个int的大小即sizeof（int）=4
      如果是一个char      ，+1的结果是增加一个char大小即sizeof（char）=1

    • 指针减整数类似加法
    数组中的应用

    由于数组中的数组元素是连续存储、类型一致。所以可以使用指针+-整数来取出数组元素
